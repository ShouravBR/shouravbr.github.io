Parallelism with multiprocessing `Pool`

Posted on Wed 16 June 2021 last updated: Wed 16 June 2021
Category: Python

The multiprocessing process makes it easy for users to achieve parallelism. The basic usage can be found on the official Python Docs [1]. In this post, we implement the multiprocessing pool in a module and use it in our "main" function.

We create the class, say "ShadowCloneJutsu", which uses the multiprocessings library and will run the @staticmethod [2] parallelly. Next, we use Pool.imap_unordered since the order of output does not matter and we can process the output as soon as it is available. Alternative to this can be found in the Python Docs [1]. Finally, we save the script as "jutsus.py"

## saved as jutsus.py
from multiprocessing import Pool

class ShadowCloneJutsu:
    def __init__(self, list_of_tasks):
        self.list_of_tasks = list_of_tasks

    @staticmethod
    def square(x):
        ''' returns square of number '''
        return x*x

    def cast(self, number_of_clones):
        with Pool(number_of_clones) as p:
            for individual_result in p.imap_unordered(
                ShadowCloneJutsu.square,
                self.list_of_tasks
                ):
                print(individual_result)

Now, we import our module and run cast function. However, we can not hastily run this function without checking the name of the scope which is stored in the __name__ variable [3]. We must check if __name__ is equal to '__main__' and then run cast. This is because, on Windows, the multiprocessing library will clone, let us call them child processes, and reexcute the main process. This means, the child processes will also run the cast function and clone recursively unless we check the scope. The child process will not be in '__main__' scope. And, checking the main scope is a good practice [4].

Here is the final code and output.

from jutsus import ShadowCloneJutsu

jutsu = ShadowCloneJutsu([1,2,3,4])

if __name__ == '__main__':
    jutsu.cast(2)
